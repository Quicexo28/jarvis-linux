Arquitectura — Pairing QR + Modo Móvil

Flujo completo desde que el PC genera el QR hasta que el celular está conectado

PC (JARVIS DESKTOP)
BACKEND · :8788
• Escucha en 0.0.0.0 (no 127.0.0.1)
• Genera token aleatorio al iniciar
• tailscale ip -4 → IP externa
NUEVOS ENDPOINTS:
GET /api/mobile/token
POST /api/mobile/auth
GET /api/mobile/status
→
FRONTEND · MODO SISTEMA
• Nueva sección "Conexión Móvil"
• Muestra QR con URL de Tailscale
• Fallback a LAN si no hay Tailscale
• Countdown 10 min + botón refrescar
• Indicador "X dispositivos conectados"
↓ QR contiene: http://<tailscale-ip>:8788?token=<token>
CELULAR · BROWSER
1 · ESCANEA QR
Abre URL en browser
App detecta ?token=xxx
POST /api/mobile/auth
→
2 · BACKEND VALIDA
Token válido → 200 OK
Token expirado → 401
Token ya usado → reutiliza sesión
→
3 · MODO MOBILE
Token → localStorage
setApiBase(tailscale-url)
Renderiza MobileClient.tsx
RECONEXIÓN AUTOMÁTICA
Si el celular cierra el browser y vuelve → token en localStorage → auto-auth → vuelve al modo mobile sin re-escanear
Si el PC reinicia → token cambia → el celular ve 401 → muestra pantalla "Re-escanea el QR"
